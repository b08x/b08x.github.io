Question: How Does Using the Word "expert" in a GPT Prompt Influence the Response GPT Provides?

When GPT is given a prompt that includes the word "expert," it is more likely to generate text that is more informative and comprehensive. This is because the word "expert" signals to GPT that the desired output is something that an expert would produce.

However, it is important to note that using the word "expert" does not guarantee that GPT will generate text that is accurate and informative. GPT is still a machine learning model, and it is therefore capable of making mistakes.

For example, when GPT is asked to write a summary of a scientific paper without including the word "expert," it is more likely to generate a summary that is more creative and engaging, not necessarily factually accurate. This is because the word "expert" can sometimes signal to GPT that the desired output is something that is dry and boring. Without the word "expert" it will by default assume that the desired output is something that should be interesting and engaging, and it will therefore use more imaginative phrasing in it's generated summary rather than a more dry recitation of facts.

A quick test to demonstrate how using the word "expert" in a GPT prompt affects the outcome:

First, I ask it to write a script without giving it any specific context:

Using GPT3.5

  • Prompt: Write a script that will check for disk usage on a given host.

  • Response:
    Certainly! Here's an example script in Python that uses the paramiko library to connect to a remote host via SSH and check the disk usage:

import paramiko

def check_disk_usage(host, username, password):
    try:
        # Create an SSH client
        client = paramiko.SSHClient()
        client.set_missing_host_key_policy(paramiko.AutoAddPolicy())

        # Connect to the remote host
        client.connect(host, username=username, password=password)

        # Execute the df command to check disk usage
        stdin, stdout, stderr = client.exec_command('df -h')

        # Print the output
        print(stdout.read().decode())

        # Close the SSH connection
        client.close()
    except paramiko.AuthenticationException:
        print('Authentication failed for the host.')
    except paramiko.SSHException as ssh_exception:
        print('Unable to establish SSH connection:', str(ssh_exception))
    except paramiko.Exception as e:
        print('An error occurred:', str(e))


hostname = 'example.com'
username = 'your_username'
password = 'your_password'

check_disk_usage(hostname, username, password)

Make sure you have the paramiko library installed before running this script. You can install it using pip: pip install paramiko.

Replace 'example.com', 'your_username', and 'your_password' with the appropriate values for your remote host. This script connects to the host using SSH, executes the df -h command to check the disk usage, and then prints the output.

Note that using a password for authentication in this script is not recommended for security reasons. It's preferable to use key-based authentication instead.

“Language Primitive”

Consider the following prompt:

I want you to act as a machine learning engineer. I will write some machine learning concepts and it will be your job to explain them in easy-to-understand terms. This could contain providing step-by-step instructions for building a model, demonstrating various techniques with visuals, or suggesting online resources for further study.

Certainly! Let's map the Cognitive Grammar elements present in the given prompt:

  1. Participants:

    • "I": Refers to the person addressing the model, indicating their intention and role as the requester.
    • "you": Refers to the model (ChatGPT), assigning it the role of a machine learning engineer.

  2. Processes:

    • "want": Expresses the mental process of desiring or intending.
    • "act as": Expresses the process of taking on a specific role or function.
    • "write": Expresses the process of composing written content.
    • "explain": Expresses the process of making concepts or information clear to others.
    • "contain": Expresses the process of including or comprising something.

  3. Circumstances:

    • "in easy-to-understand terms": Specifies the manner or quality of the explanation, indicating the desired level of simplicity and clarity.

  4. Objects:

    • "machine learning engineer": Refers to the desired role or position that the requester wants the model to adopt.

These Cognitive Grammar elements help us understand the cognitive processes, roles, actions, and specifications involved in the prompt. They provide insights into the mental and linguistic operations required for the requester and the model to engage in a conversation about machine learning concepts, explanations, and the desired style of communication.
