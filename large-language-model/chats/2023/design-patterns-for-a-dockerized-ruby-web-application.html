- name: Starting docker tasks  
  ansible.builtin.debug:  
    msg: "Starting docker tasks"  
  
- name: Install docker packages  
  package:  
    name: "{{ docker.packages }}"  
    state: present  
  become: yes  
  when: ansible_os_family == "Debian" or ansible_os_family == "RedHat"  
  
- name: Install docker packages (Arch)  
  aur:  
    use: auto  
    name: "{{ docker.packages }}"  
    state: present  
  become_user: "{{ user.name }}"  
  when: ansible_os_family == "Archlinux"  
  notify: docker service  
  tags: ['packages']  
  
- name: Add user to docker group  
  ansible.builtin.user:  
    name: "{{ user.name }}"  
    groups: docker  
    append: yes  
  check_mode: no  
  ignore_errors: yes  
  
- name: Set docker storage location  
  ansible.builtin.lineinfile:  
    path: "/usr/lib/systemd/system/docker.service"  
    regexp: "^ExecStart=/usr/bin/dockerd -H fd://"  
    line: "ExecStart=/usr/bin/dockerd --data-root {{ docker.storage }} -H fd:// --containerd=/run/containerd/containerd.sock"  
  notify: reload systemd daemon  
  
- name: Create docker config directory  
  ansible.builtin.file:  
    path: /etc/docker  
    state: directory  
    owner: root  
    group: root  
    mode: '0755'  
  
- name: Enable experimental features in docker  
  ansible.builtin.copy:  
    content: |  
      {  
        "experimental": true  
      }  
  
    dest: "/etc/docker/daemon.json"  
    owner: root  
  
- name: Disable overlay redirect  
  ansible.builtin.copy:  
    content: |  
      options overlay metacopy=off redirect_dir=off  
  
    dest: "/etc/modprobe.d/disable-overlay-redirect-dir.conf"  
    owner: root  
  

- name: Reboot host  
  ansible.builtin.reboot:  
  async: 1  
  poll: 0  
  ignore_errors: True  
  when: reboot is defined  
  
- name: Wait for host to reboot  
  ansible.builtin.wait_for_connection:  
    connect_timeout: 20  
    sleep: 5  
    delay: 5  
    timeout: 120  
  when: reboot is defined

ask Engineer: What would be the pros and cons of running docker-compse as an ansible task

03:56:25 PM-assistant

🧑‍🔧 Engineer: Great question. Running Docker Compose as an Ansible task can have several advantages and disadvantages:

Pros:

  1. Simplicity: Docker Compose simplifies the process of managing multi-container Docker applications. You can define all your services in a single YAML file and start them all with a single command. This can make your Ansible tasks simpler and easier to understand.

  2. Consistency: By using Docker Compose, you can ensure that your application is always deployed in the same way, regardless of the environment. This can help to eliminate "it works on my machine" problems.

  3. Scalability: Docker Compose makes it easy to scale your application by simply changing a number in your Compose file. This can be very useful if you need to quickly scale up or down in response to changes in demand.

Cons:

  1. Complexity: While Docker Compose can simplify the management of Docker containers, it also adds another layer of complexity to your deployment process. You now have to manage both your Ansible tasks and your Docker Compose files.

  2. Dependency: By using Docker Compose, you're adding another dependency to your deployment process. If Docker Compose is not available or not working correctly, it could prevent your application from being deployed.

  3. Limited Control: Docker Compose abstracts away many of the details of managing Docker containers. While this can make things simpler, it also means you have less control over the individual containers. If you need to perform complex or unusual tasks, you may find that Docker Compose is not flexible enough.

In general, whether or not to use Docker Compose as an Ansible task will depend on the specific needs of your project. If the simplicity and consistency of Docker Compose outweigh the added complexity and dependency, then it could be a good choice.
