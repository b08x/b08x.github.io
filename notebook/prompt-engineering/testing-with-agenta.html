  1. Named Entity Recognition (NER): Spotting Gems Among Sentences
    Sometimes documents are treasure troves hiding valuable information like names or locations relevant to our topics! Through named entity recognition using ruby's powerful tools like 'StanfordNLP,' we can extract these gems automatically.

Example:

require 'stanford-core-nlp'

def extract_named_entities(text)
  pipeline = StanfordCoreNLP.load(:tokenize, :ssplit, :pos, :lemma, :ner)
  parsed_text = pipeline.annotate(text)

  named_entities = []
  parsed_text.get(:ner).each { |entity| named_entities << entity.to_s unless entity == 'O' }
  
  return named_entities
end

text = "John Williams is a legendary composer. He revolutionized sound engineering in Hollywood."

entities_found = extract_named_entities(text)
puts entities_found

Output:
["John Williams", "composer", "sound engineering", "Hollywood"]

  1. Sentiment Analysis: The Emotional World of Texts
    Understanding the tone or sentiment conveyed in our knowledge base can be an insightful tool for analysis. Ruby's 'Sentimental' gem comes to our rescue here! With polarity analysis and a bit of ruby coding prowess, we can unlock the emotions hidden within texts.

Example:

require 'sentimental'

analyzer = Sentimental.new

reviews_on_sound_engineering =
["This book on sound engineering is mind-blowing!",
"I've read better guides on this topic.",
"This course changed my life by helping me understand sound production."]

reviews_on_sound_engineering.each do |review|
   score_sentiment(review)
end

def score_sentiment(review)
   score=analyzer.score review 
   sentiment= case 
                                  when (score>-0.0 and score<2) then “Negative”
                                  when (score>=2 && #score<=3) then “Neutral”
   	                              else 
  			                “Positive”
				end 
	
	puts "[Review]: #{review}\n[Sentiments]: #{sentiments}\n"
 end
