RubyGem Tomoto Documentation

The tomoto gem is a Ruby binding for the Tomoto topic modeling library, which follows the tomotopy API. This gem allows you to perform topic modeling tasks in Ruby, such as Latent Dirichlet Allocation (LDA).

Installation

Add this line to your application's Gemfile:

bundle add 'tomoto'

And then execute:

$ bundle install

Or install it yourself as:

$ gem install tomoto

Usage

Basic LDA Model

Here's a basic example of how to use the LDA model:

require 'tomoto'

# Create an LDA model with 20 topics
lda = Tomoto::LDA.new(k: 20)

# Add documents to the model
lda.add_doc("This is the first document")
lda.add_doc("This is the second document")
lda.add_doc("And this is the third one")

# Train the model
lda.train(100)  # Train for 100 iterations

# Get the topics
topics = lda.get_topic_words(top_n: 10)
topics.each_with_index do |topic, i|
  puts "Topic #{i}: #{topic.join(', ')}"
end

Advanced Usage

Creating a Model

You can create different types of topic models:

# LDA model
lda = Tomoto::LDA.new(k: 20)

# Hierarchical Dirichlet Process (HDP) model
hdp = Tomoto::HDP.new(initial_k: 20)

# Pachinko Allocation Model (PAM)
pam = Tomoto::PAM.new(k: 20, t: 5)

Adding Documents

You can add documents in various ways:

# Add a single document
lda.add_doc("This is a document")

# Add a document with custom weights
lda.add_doc(["word1", "word2", "word3"], [1.0, 2.0, 0.5])

# Add multiple documents at once
docs = ["Doc 1", "Doc 2", "Doc 3"]
lda.add_docs(docs)

Training the Model

Train the model with a specified number of iterations:

lda.train(100)  # Train for 100 iterations

# You can also train incrementally
lda.train(50)
lda.train(50)  # This will continue from where the previous training left off

Retrieving Topics

Get the top words for each topic:

topics = lda.get_topic_words(top_n: 10)
topics.each_with_index do |topic, i|
  puts "Topic #{i}: #{topic.join(', ')}"
end

Document-Topic Distribution

Get the topic distribution for a specific document:

doc_topics = lda.get_doc_topic_dist(0)  # Get topics for the first document
doc_topics.each_with_index do |weight, topic_id|
  puts "Topic #{topic_id}: #{weight}"
end

Inference on New Documents

Perform inference on new, unseen documents:

new_doc = "This is a new unseen document"
doc_topics = lda.infer(new_doc)

Saving and Loading Models

Save your trained model:

lda.save("my_lda_model.bin")

Load a previously saved model:

loaded_lda = Tomoto::LDA.load("my_lda_model.bin")

Model Parameters

You can access and modify various model parameters:

# Get the number of topics
puts lda.k

# Get the alpha parameter
puts lda.alpha

# Set the alpha parameter
lda.alpha = 0.1

# Get the total number of words in the corpus
puts lda.num_words

# Get the total number of documents
puts lda.num_docs

The ruby-tomoto gem follows the tomotopy API, so the inference process is similar to that of tomotopy. Here's how you can perform inference using ruby-tomoto:

Loading a Pre-trained Model

First, you need to load a previously trained model. Assuming you have already trained and saved an LDA model, you can load it like this:

require 'tomoto'

# Load the saved model
lda = Tomoto::LDA.load("my_lda_model.bin")
  1. Prepare the new document:
    For inference, you need to prepare the new, unseen document. This usually involves tokenizing the text into words:
new_doc = "This is a new unseen document about some topic"
tokens = new_doc.split  # Simple whitespace tokenization
  1. Perform inference:
    Now you can use the loaded model to infer the topic distribution for the new document:
# Infer topic distribution for the new document
doc_topic_dist = lda.infer(tokens)

The infer method returns an array representing the topic distribution for the given document. Each element in the array corresponds to the probability of the document belonging to that topic.

  1. Interpret the results:
    You can now interpret the inference results. For example, you might want to find the most probable topic for this document:
most_probable_topic = doc_topic_dist.each_with_index.max_by { |prob, _| prob }[1]
puts "The most probable topic for this document is Topic #{most_probable_topic}"

# Print the full topic distribution
doc_topic_dist.each_with_index do |prob, topic_id|
  puts "Topic #{topic_id}: #{prob}"
end
  1. Get the top words for the most probable topic:
    To better understand what the inferred topic represents, you can get the top words for that topic:
top_words = lda.get_topic_words(most_probable_topic, top_n: 10)
puts "Top words for the most probable topic: #{top_words.join(', ')}"

Here's a complete example putting it all together:

require 'tomoto'

# Load the saved model
lda = Tomoto::LDA.load("my_lda_model.bin")

# New document for inference
new_doc = "This is a new unseen document about artificial intelligence and machine learning"
tokens = new_doc.split

# Perform inference
doc_topic_dist = lda.infer(tokens)

# Find the most probable topic
most_probable_topic = doc_topic_dist.each_with_index.max_by { |prob, _| prob }[1]
puts "The most probable topic for this document is Topic #{most_probable_topic}"

# Print the full topic distribution
doc_topic_dist.each_with_index do |prob, topic_id|
  puts "Topic #{topic_id}: #{prob}"
end

# Get top words for the most probable topic
top_words = lda.get_topic_words(most_probable_topic, top_n: 10)
puts "Top words for the most probable topic: #{top_words.join(', ')}"

This script loads a pre-trained LDA model, performs inference on a new document, identifies the most probable topic, prints the full topic distribution, and shows the top words for the most probable topic.

Remember that the quality of inference depends on how well your model was trained and how similar the new document is to the training corpus. If the new document contains words or concepts not seen during training, the inference might be less accurate.

Integration with Jongleur

To integrate tomoto with Jongleur for topic modeling within a workflow, you can create a TopicModelerTask like this:

require 'tomoto'

class TopicModelerTask < Jongleur::WorkerTask
  def execute
    begin
      processed_sentences = JSON.parse(@@redis.get("processed_sentences"))
      
      # Initialize Tomoto LDA model
      lda = Tomoto::LDA.new(k: 20)  # 20 topics
      
      # Add documents to the model
      processed_sentences.each do |sentence|
        lda.add_doc(sentence)
      end
      
      # Train the model
      lda.train(100)  # 100 iterations
      
      # Get the top 10 words for each topic
      topics = lda.get_topic_words(top_n: 10)
      
      # Format topics for output
      formatted_topics = topics.map.with_index do |topic, index|
        {
          id: index,
          words: topic
        }
      end
      
      @@redis.set("topics", formatted_topics.to_json)
    rescue => e
      logger.error "Error in TopicModelerTask: #{e.message}"
      raise
    end
  end
end

This task can be integrated into your Jongleur workflow to perform topic modeling on processed text data.

Remember to handle exceptions appropriately and adjust the parameters (number of topics, iterations, etc.) based on your specific use case and dataset.
