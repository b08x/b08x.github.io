Flowbots

This document outlines an optimized architecture for the Flowbots project, focusing on the efficient integration of Nanobots and Jongleur for AI-powered workflow automation.

Objective

To create a robust and scalable system for managing and executing complex, stateful multi-actor agent workflows using Nanobots and Jongleur.

Architecture Overview

The system leverages a microservices architecture, with each component containerized using Docker for easy deployment and scalability.

  • Nano Bots Registry: A centralized repository hosting individual Nanobots, each defined as a YAML file encapsulating its functionality and dependencies.
  • Advanced Text Processing Orchestrator: This component receives input text and orchestrates the workflow, utilizing Jongleur for task scheduling and management.
  • Jongleur: Manages the execution of tasks defined within the workflow, ensuring dependencies are met and resources are allocated efficiently.
  • Preprocessing Task: Utilizes NLP libraries (SpaCy, Pragmatic Tokenizer) for text preprocessing, preparing it for subsequent tasks.
  • Topic Modeling Task: Employs a topic modeling algorithm to extract key themes and concepts from the preprocessed text.
  • Language Model Task: Leverages pre-trained language models, potentially accessed through FlowiseAI, to generate responses or perform specific actions based on the processed information.

Workflow

  1. Workflow Definition: Workflows are defined within the Advanced Text Processing Orchestrator, outlining the sequence of tasks to be executed by Jongleur.
  2. Task Execution: Jongleur manages the execution of tasks according to the defined workflow, leveraging Nanobots from the registry as needed.
  3. Data Flow: Data is passed between tasks as defined in the workflow. Intermediate results can be persisted in Redis for efficient retrieval.
  4. LLM Interaction (Optional): For tasks requiring language models, the Language Model Task interacts with the appropriate Nanobot
  5. Output Generation: The final output of the workflow is generated based on the processed information and the objectives defined in the workflow.

Next Steps

  • Nanobot Development: Design and implement Nanobots for specific functionalities, ensuring they adhere to the defined specification for seamless integration.
  • Workflow Orchestration: Define workflows within the Advanced Text Processing Orchestrator, specifying the sequence of tasks, data flow, and dependencies.
  • Jongleur Configuration: Configure Jongleur for optimal task scheduling, resource allocation, and error handling.
  • Integration Testing: Thoroughly test the integrated system, ensuring data flows correctly between components and workflows execute as expected.

#!/usr/bin/env ruby
# frozen_string_literal: true

lib_dir = File.expand_path(File.join(__dir__, '..', 'lib'))
$LOAD_PATH.unshift lib_dir unless $LOAD_PATH.include?(lib_dir)

require 'nano-bots'
require 'jongleur'
require 'redis'
require 'json'
require 'yaml'

require "dotenv"
Dotenv.load(File.join(__dir__, "..", "docker", ".env"))

require "utils/logging"

include Logging

class Jongleur::WorkerTask
  @@redis = Redis.new(host: "localhost", port: 6379, db: 15)
end

class WorkflowAgent
  def initialize(role, cartridge_file)
    @role = role
    @state = {}
    @bot = NanoBot.new(
      cartridge: cartridge_file
    )
  end

  def process(input)

    @bot.eval(input) do |content, fragment, finished, meta|
      @response = content unless content.nil?
      # print fragment unless fragment.nil?
    end

    update_state(@response)
    @response
  end

  def save_state
    Jongleur::WorkerTask.class_variable_get(:@@redis).hset(
      Process.pid.to_s,
      "agent:#{@role}",
      @state.to_json
    )
  end

  def load_state
    state_json = Jongleur::WorkerTask.class_variable_get(:@@redis).hget(
      Process.pid.to_s,
      "agent:#{@role}"
    )
    @state = JSON.parse(state_json) if state_json
  end

  private

  def update_state(response)
    @state[:last_response] = response
  end
end

class ResearcherTask < Jongleur::WorkerTask
  def execute
    begin
      agent = WorkflowAgent.new("researcher", "researcher.yml")
      agent.load_state
      result = agent.process("Research on Advancements in Garden Gnome Technology")
      agent.save_state
      @@redis.set("research_result", result.to_json)
    rescue => e
      logger.error "Error in ResearcherTask: #{e.message}"
      raise
    end
  end
end

class WriterTask1 < Jongleur::WorkerTask
  def execute
    begin
      agent = WorkflowAgent.new("writer1", "writer.yml")
      agent.load_state
      research_result = JSON.parse(@@redis.get("research_result"))
      article = agent.process("Write an article based on: #{research_result}")
      agent.save_state
      @@redis.rpush("articles", article)
    rescue => e
      logger.error "Error in WriterTask1: #{e.message}"
      raise
    end
  end
end

class WriterTask2 < Jongleur::WorkerTask
  def execute
    begin
      agent = WorkflowAgent.new("writer2", "writer.yml")
      agent.load_state
      research_result = JSON.parse(@@redis.get("research_result"))
      article = agent.process("Write a fantasy-horror narrative based on: #{research_result}")
      agent.save_state
      @@redis.rpush("articles", article)
    rescue => e
      logger.error "Error in WriterTask2: #{e.message}"
      raise
    end
  end
end

class FinalizeTask < Jongleur::WorkerTask
  def execute
    begin
      agent = WorkflowAgent.new("finalizer", "editor.yml")
      articles = @@redis.lrange("articles", 0, -1)
      final_result = agent.process("Summarize and finalize these articles: #{articles.join('\n\n')}")
      puts "Final Result:"
      puts final_result
    rescue => e
      logger.error "Error in FinalizeTask: #{e.message}"
      raise
    end
  end
end

class WorkflowOrchestrator
  def initialize
    @agents = {}
  end

  def add_agent(role, cartridge_file)
    @agents[role] = WorkflowAgent.new(role, cartridge_file)
  end

  def define_workflow(workflow_definition)
    Jongleur::API.add_task_graph(workflow_definition)
  end

  def run_workflow
    Jongleur::API.print_graph('/tmp')

    Jongleur::API.run do |on|
      on.completed do |task_matrix|
        puts "Workflow completed"
        puts task_matrix
      end
    end
  end
end

# Example usage
orchestrator = WorkflowOrchestrator.new
orchestrator.add_agent('researcher', 'researcher.yml')
orchestrator.add_agent('writer1', 'writer.yml')
orchestrator.add_agent('writer2', 'writer.yml')
orchestrator.add_agent('finalizer', 'editor.yml')

workflow_graph = {
  ResearcherTask: [:WriterTask1, :WriterTask2],
  WriterTask1: [:FinalizeTask],
  WriterTask2: [:FinalizeTask]
}

orchestrator.define_workflow(workflow_graph)
orchestrator.run_workflow

Implementation Plan

This document outlines a comprehensive DevOps implementation plan for the Flowbots project, leveraging Nanobots, Jongleur, Docker, and industry-standard tools for efficient and scalable AI-powered workflow automation.

Project Goal: To create a robust, scalable, and automated system for managing and executing complex, stateful, multi-actor agent workflows.

Phase 1: Architecture and Design

  1. Define Workflow Components:
    • Nano Bots: Lightweight, specialized AI-powered bots defined in YAML. Responsible for individual tasks within a workflow.
      • Design and implement Nanobots for specific functionalities (research, writing, editing).
      • Ensure Nanobots adhere to a defined specification for seamless integration with the orchestrator.
    • Jongleur: Workflow management framework handling process scheduling, execution, dependencies, and parallel execution.
      • Configure Jongleur for optimal task scheduling, resource allocation, and error handling.
    • Redis: In-memory data store for efficient state management and data sharing between workflow stages.
    • Advanced Text Processing Orchestrator: Receives input, orchestrates the workflow, manages data flow, and generates the final output.
      • Define workflows within the orchestrator, specifying task sequences, data flow, and dependencies.
  2. Design Data Flow and State Management:
    • Utilize Redis for:
      • Storing intermediate results from Nanobots.
      • Enabling stateful interactions between workflow stages.
  3. Implement NLP Processing Pipeline:
    • Preprocessing Task: Prepare text for analysis using:
      • PragmaticTokenizer: Tokenization.
      • ruby-spacy: Part-of-speech tagging and morphology analysis.
    • Topic Modeling Task (Optional): Extract key themes and concepts using a topic modeling algorithm.
    • Language Model Task (Optional): Integrate with pre-trained language models via FlowiseAI for advanced language processing.
  4. Design Evaluation and Feedback Loop:
    • Integrate automatic evaluation tools to assess NLP prompt quality.
    • Incorporate mechanisms for human feedback.
    • Implement dynamic adjustment of agent behavior based on feedback.

Phase 2: Development and Containerization

  1. Develop Nanobots:
    • Create YAML files for each Nanobot, specifying:
      • meta: Descriptive information (name, author, version, etc.).
      • behaviors: Instructions and directives for the bot's actions.
      • interfaces: Input/output formats and prompts.
      • provider: Underlying AI service (e.g., Cohere, Google).
  2. Develop Orchestrator Logic:
    • Implement workflow definition and execution logic using Jongleur.
    • Handle data flow between tasks and persistence in Redis.
    • Manage Nanobot selection and execution based on workflow requirements.
  3. Containerize with Docker:
    • Create Dockerfiles for:
      • Nanobots Registry: A simple web server using Rack to serve Nanobot YAML files.
      • Advanced Text Processing Orchestrator: Includes Ruby, gems, NLP dependencies (SpaCy), and the orchestrator script.
    • Define a Docker Compose file for local development and testing, orchestrating the registry, orchestrator, and Redis.

Phase 4: Monitoring, Logging, and Continuous Improvement

  1. Implement Monitoring:
    • Graylog: Monitor resource usage of containers.
    • Langfuse: Implement tracing for performance analysis.
  2. Implement Logging:
    • Centralize logs from all components for easier debugging and analysis.
  3. Security:
    • Implement authentication and authorization for the Nanobots Registry and API endpoints.
  4. Continuous Improvement:
    • Regularly review logs and monitoring data to identify bottlenecks and areas for optimization.
    • Implement performance tuning and resource optimization strategies.
    • Continuously update Nanobots and workflows based on feedback and evolving requirements.

Conclusion

This DevOps plan provides a roadmap for building and managing the Flowbots project. By focusing on modularity, automation, and continuous improvement, you can create a robust and efficient system for AI-powered workflow automation.
