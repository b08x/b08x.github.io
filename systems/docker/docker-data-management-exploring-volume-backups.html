Backing Up Docker Volumes: Protecting Your Persistent Data

While Docker makes it easy to create, share, and deploy applications, ensuring the persistence and recoverability of your data requires a dedicated strategy for backing up Docker volumes.

Common Methods for Backing Up Docker Volumes

  1. Using a Dedicated Backup Tool:

    • Advantages: Simplifies the backup and restore process, often with scheduling and automation features.
    • Tools:
      • Docker Volume Plugins: Plugins like "vieux/sshfs" allow mounting remote directories as volumes, enabling backups to external systems.
      • Third-party Backup Solutions: Specialized backup solutions designed for containerized environments offer features tailored for Docker volumes.

  2. Manual Backup and Restore:

    • Advantages: Provides flexibility and control over the backup process.

    • Process:

      1. Create a temporary container with access to the volume.
      2. Use tar, cp, or other tools to copy the volume data to a backup location (local or remote).
      3. For restoration, reverse the process, copying data back into a new or existing volume.

      Example:

      # Backup
docker run --rm -v my-volume:/data -v $(pwd):/backup busybox tar cvf /backup/my-volume-backup.tar /data

# Restore
docker run --rm -v my-volume:/data -v $(pwd):/backup busybox tar xvf /backup/my-volume-backup.tar -C /data

  3. Host-Level Backups:

    • Advantages: Simple if your Docker volumes reside on the host filesystem.
    • Process: Use your preferred host-level backup solution (snapshots, file-level backups) to back up the directory where Docker volumes are stored.
    • Caveat: Requires careful management to avoid backing up unnecessary data and ensure data consistency.

Best Practices

  • Regular Backups: Establish a regular backup schedule based on your data update frequency and recovery time objectives (RTO).
  • Offsite Storage: Store backups in a different location from your Docker host to protect against data loss due to physical disasters.
  • Test Restorations: Periodically test your backup and restore procedures to ensure data integrity and recoverability.
  • Document the Process: Maintain clear documentation of your backup strategy, including tools used, backup locations, and restoration steps.

By implementing a robust Docker volume backup strategy, you can ensure the safety and availability of your critical data, even in the face of unexpected events.

#!/bin/bash

# ------------------------------------------------------------------------------
# Script Name: docker-volume-backup.sh
# Description: This script provides an interactive way to back up and restore
#              Docker volumes using the 'gum' tool for enhanced UI.
#
# Requirements:
#   - gum: Install from https://github.com/charmbracelet/gum
#   - fd:  Install from https://github.com/sharkdp/fd
#   - sd:  Install from https://github.com/chmln/sd
#
# Usage:
#   - Backup all volumes:
#       ./docker-volume-backup.sh backup
#
#   - Backup specific volumes:
#       ./docker-volume-backup.sh backup volume1 volume2 ...
#
#   - Restore volumes (interactive selection from backups):
#       ./docker-volume-backup.sh restore
# ------------------------------------------------------------------------------

# --- Configuration ------------------------------------------------------------

# Backup directory (change if needed)
backup_dir="/backups/docker-volumes"

# Array of all available volumes (update as needed)
all_volumes=(
  "chromadb-data-volume"
  "docker_chroma_persistent_folder"
  "docker_postgresql_data"
  "docker_postgresql_local_data"
  "docker_redis_data"
  "flowise-chroma-data"
  "flowise-pgvector-data"
  "flowise-redis-data"
  "sillytavern-config-volume"
  "sillytavern-data-volume"
  "sillytavern-extensions-volume"
  "sillytavernextras-cache-volume"
)

# --- Functions ---------------------------------------------------------------

# Function to backup Docker volumes
# Arguments: (optional) List of volumes to back up. If none are provided,
#            all volumes in the 'all_volumes' array will be backed up.
function backup_volumes {
  local volumes_to_backup
  local backup_name

  # Use gum choose for interactive volume selection (if no arguments)
  if [[ $# -eq 0 ]]; then
    volumes_to_backup=$(gum choose "${all_volumes[@]}" --no-limit)
  else
    volumes_to_backup=("$@")
  fi

  # Use gum input for backup name
  backup_name=$(gum input --placeholder "Enter a name for this backup" --value "backup_$(date +%Y-%m-%d_%H-%M-%S)")

  local backup_path="$backup_dir/$backup_name"
  mkdir -p "$backup_path"

  # Backup each selected volume
  for volume in $volumes_to_backup; do
    echo "Backing up volume: $volume to $backup_path"
    docker run --rm -v "$volume":/volume -v "$backup_path":/backup alpine tar cvf "/backup/${volume}.tar" /volume
  done
  echo "Backup complete. Backups stored in: $backup_path"
}

# Function to restore Docker volumes
function restore_volumes {
  local backup_name
  local volumes_to_restore

  # Use gum input for backup name
  backup_name=$(gum input --placeholder "Enter the name of the backup to restore from")

  local backup_path="$backup_dir/$backup_name"
  if [[ ! -d "$backup_path" ]]; then
    echo "Error: Backup directory not found: $backup_path"
    exit 1
  fi

  # Get available volumes in the backup directory using fd and sd
  available_volumes=$(fd -t f -e .tar "$backup_path" | sd '.tar$' '')

  # Use gum filter for interactive volume selection from available backups
  volumes_to_restore=$(echo "$available_volumes" | gum filter --placeholder "Filter volumes to restore")

  if [[ -z "$volumes_to_restore" ]]; then
    echo "No volumes selected for restoration."
    exit 0
  fi

  # Restore each selected volume
  for vol_to_restore in $volumes_to_restore; do
    backup_file="$backup_path/${vol_to_restore}.tar"
    if [[ -f "$backup_file" ]]; then
      echo "Restoring volume: $vol_to_restore from $backup_file"
      docker run --rm -v "$vol_to_restore":/volume -v "$backup_path":/backup alpine tar xvf "/backup/${vol_to_restore}.tar" -C /volume
    else
      echo "Error: Backup file not found: $backup_file"
    fi
  done
}

# --- Main Script Execution ----------------------------------------------------

# Check for command line arguments
if [[ $# -eq 0 ]]; then
  # No arguments: Interactive mode
  echo "Choose an action:"
  echo "  1. Backup volumes"
  echo "  2. Restore volumes"
  choice=$(gum choose --cursor-prefix ">" --selected-prefix ">"  "Backup volumes" "Restore volumes")

  case "$choice" in
    "Backup volumes")
      backup_volumes
      ;;
    "Restore volumes")
      restore_volumes
      ;;
    *)
      echo "Invalid choice. Exiting."
      exit 1
      ;;
  esac
else
  # Arguments provided: Non-interactive mode
  action="$1"
  shift # Remove action from arguments

  case "$action" in
    backup)
      backup_volumes "$@"
      ;;
    restore)
      restore_volumes 
      ;;
    *)
      echo "Invalid action: $action. Choose either 'backup' or 'restore'."
      exit 1
      ;;
  esac
fi
