KnowledgeGraphs

A Knowledge Graph is a collection of Entities, Entity Types, and Entity Relationship Types that manifests as an intelligible Web of Data informed by an Ontology.

Knowledge Graphs are created by describing Entities and Entity Relationship Types using sentences that are deployed using Linked Data principles.

To construct the prompt and reference specific parts of the knowledge graph in the response, you can use placeholders or variables in the prompt text. These placeholders will represent the different components of the knowledge graph, allowing the AI language model to provide responses that correspond to the referenced skills. Here's how you can do it:

  1. Define Placeholder Variables: Assign a unique placeholder variable to each skill component in the knowledge graph. For example, you can use curly braces {} and a variable name for each skill. For clarity, you can use the same variable names as the "id" values in the knowledge graph.

  2. Incorporate Placeholders in the Prompt: In the prompt text, use the defined placeholder variables to reference specific skills or sub-skills. You can ask the model questions or instruct it to provide information related to those skills. Be sure to use the same format for the placeholders as used in the knowledge graph.

  3. Map Placeholder Variables in the Response: When processing the AI's response, you can map the placeholders in the prompt to the corresponding parts of the knowledge graph. This will help you understand which skill or component the AI is addressing in its answer.
