blueprints-cli

A CLI for managing and generating code from blueprints.

About

`blueprints-cli` is a powerful command-line tool designed to streamline your development workflow. It allows you to create, manage, and use 'blueprints' - reusable code templates. With built-in AI assistance, it can help generate names, descriptions, and even suggest improvements for your blueprints. This tool is perfect for developers looking to standardize project scaffolding and accelerate development.

Key Features

  • Blueprint Management: Easily create, list, view, edit, and delete blueprints.
  • AI-Powered Generation: Use AI to generate blueprint names, descriptions, and categories.
  • Vector-based Search: Find relevant blueprints using semantic search.
  • Export & Share: Export blueprints to share with your team or the community.
  • Interactive CLI: An enhanced interactive menu for easy navigation and command execution.
  • Extensible Provider Support: Integrates with LLM providers like Ollama and OpenRouter.

Powered by Sublayer

The AI capabilities of `blueprints-cli` are built on the Sublayer framework. Sublayer provides a robust foundation for creating sophisticated AI agents and tools, handling complex interactions with LLMs for tasks like generation and search.

Architecture Overview

Blueprint Storage Flow

1. User Input

User submits a new blueprint via `blueprint submit`.

2. Embedding Generation

The blueprint content is sent to an embedding provider (e.g., Ollama) to create a vector.

3. Data Persistence

The blueprint metadata and its vector embedding are stored in the PostgreSQL database with the pgvector extension.

Blueprint Retrieval Flow

1. User Search

User searches for blueprints using `blueprint search "query"`.

2. Query Embedding

The search query is also converted into a vector embedding using the same provider.

3. Vector Similarity Search

A cosine similarity search is performed in the database to find the closest matching blueprint vectors.

4. Results

The most relevant blueprints are returned to the user in the CLI.
