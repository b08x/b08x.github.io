# Codebase Overview: Prosodic Text Converter

The Prosodic Text Converter is a Ruby application that transforms text into natural-sounding Speech Synthesis Markup Language (SSML) using Large Language Models (LLMs) and audio analysis. It supports multiple LLM providers, audio analysis backends, and offers both command-line (CLI) and interactive terminal (TUI) interfaces. The architecture is modular, organized within lib/prosodic-text-converter/ into audio, analysis, text, and conversion modules.

High-Level Architecture:

  • Core Logic (Converter, SSMLFormatter): Manages text-to-SSML conversion, integrating linguistic analysis and LLM interaction.
  • Audio Processing (SpectrogramGenerator, SpectrogramAnalyzer, PitchAnalyzerFactory): Handles audio file conversion (SoX), spectrogram generation (MiniMagick), and pitch analysis (Aubio, Sonic Annotator).
  • LLM Integration (LLMConverter, SpeechPatternRewriter): Uses ruby_llm for multi-provider LLM support (OpenAI, Anthropic, Google Gemini, Ollama), generating prosodic markup and rephrasing text based on Systemic Functional Linguistics (SFL) principles.
  • Speech Synthesis (SpeechSynthesizer, ElevenLabsFormatter): Integrates with external APIs like ElevenLabs for generating final audio, managing SSML and voice selection.
  • Text Analysis (TextAnalyzer): Provides linguistic context using lingua and pragmatic_tokenizer for sentence segmentation, tokenization, and syllable counting.
  • Configuration (Config): Manages settings from defaults.yml, environment variables, and CLI arguments via tty-config.
  • User Interfaces (CLI, TUI): Offers flexible interaction methods.
  • Deployment: Supported by Docker for consistent execution environments.

Key Strengths:

  • Extensible Backend Support: Employs a factory pattern for pitch analysis and the ruby_llm gem for flexible LLM provider integration.
  • Robust Third-Party Integration: Orchestrates SoX, Aubio, Sonic Annotator, MiniMagick, and ElevenLabs API for comprehensive audio, image, and speech synthesis.
  • Advanced Prosody Extraction & Rewriting: Features audio-guided prosodic pattern extraction and intelligent text rewriting using LLMs based on SFL principles for nuanced speech synthesis.
  • Layered Configuration: Provides a clear and flexible configuration hierarchy.
  • Comprehensive NLP: Leverages lingua and pragmatic_tokenizer for foundational linguistic processing.

Identified Areas for Improvement:

  • Security Vulnerabilities: Critical command injection and path traversal vulnerabilities due to insufficient input sanitization. Potential XXE risk from inconsistent XML parsing options.
  • LLM Security & Robustness: Inherent prompt injection risk; mitigations need enhancement (e.g., sophisticated semantic similarity, input/output filtering).
  • Resource Management: Potential for resource exhaustion and high LLM costs with large inputs, requiring explicit limits and cost-aware strategies.
  • Feature Completeness: Sophisticated analysis methods in SpeechPatternExtractor are placeholders, limiting prosodic pattern analysis depth.
  • Dependency Management: A formal process for continuous updates, vulnerability scanning, and documentation of security implications is needed.
  • Configuration Security: Dotenv.load(overwrite: true) behavior may lead to unintended overwrites of production environment variables.

# Backlog of Tasks

1. Implement Shellwords.shellescape

High

Apply Shellwords.shellescape to user-controlled inputs passed to external shell commands in sonic-annotator and sox calls.

  • Affected: SonicAnnotatorPitchAnalyzer, SpectrogramGenerator
  • Effort: Small

2. Canonicalize and Restrict Paths

High

Implement Pathname#cleanpath, File.absolute_path, and File.expand_path for user-supplied paths.

  • Affected: CLI, SpectrogramAnalyzer, SpeechSynthesizer, TUI
  • Effort: Medium

3. Review Dotenv.load Behavior

Medium

Re-evaluate Dotenv.load(overwrite: true) usage to prioritize existing environment variables.

  • Affected: Config class, deployment scripts
  • Effort: Small

4. Verify XXE Prevention

Medium

Ensure external entity resolution is explicitly disabled for untrusted SSML content in Nokogiri calls.

  • Affected: SSMLFormatter, ElevenLabsFormatter
  • Effort: Small

5. Enhance Prompt Injection Mitigations

High

Upgrade validate_meaning_preservation with sophisticated semantic similarity models and input/output filtering.

  • Affected: LLMConverter, SpeechPatternRewriter
  • Effort: Large

6. Implement Explicit Resource Limits

Medium

Introduce explicit size/duration limits for audio files and spectrograms to prevent resource exhaustion.

  • Affected: SpectrogramGenerator, SpectrogramAnalyzer, Converter
  • Effort: Medium

7. Establish Dependency Security Management

High

Formulate a policy for regularly updating dependencies and integrate automated vulnerability scanning.

  • Affected: Entire codebase, CI/CD pipeline
  • Effort: Large (ongoing)

8. Fully Implement Sophisticated Analysis

High

Develop robust signal processing/ML algorithms for placeholder analysis methods in SpeechPatternExtractor.

  • Affected: SpeechPatternExtractor
  • Effort: Large

9. Implement LLM Token-Based Cost Estimation

Medium

Introduce mechanisms for token-based cost estimation and early exit strategies for LLM calls.

  • Affected: LLMConverter, SpeechPatternRewriter
  • Effort: Medium

# User Stories

Security Engineer

As a Security Engineer, I want all external command executions to properly escape user-controlled inputs, so that I can prevent command injection vulnerabilities.

Developer

As a Developer, I want all user-supplied file paths to be canonicalized and restricted to a secure base directory, so that the application is protected against path traversal attacks.

Operations Engineer

As an Operations Engineer, I want sensitive environment variables in production to take precedence over .env file settings, so that configuration is secure and predictable.

Security Engineer

As a Security Engineer, I want all XML/SSML parsing to explicitly disable external entity resolution, so that the application is protected from XXE vulnerabilities.

User

As a User, I want LLM interactions to be robustly filtered for malicious inputs and outputs, so that I receive safe and reliable responses and my data is protected.

System Administrator

As a System Administrator, I want explicit size and duration limits on audio and spectrogram files, so that system resources are protected from exhaustion and costs are managed.

Security Lead

As a Security Lead, I want all project dependencies to be regularly scanned for vulnerabilities and kept up-to-date, with comprehensive documentation, so that our software supply chain remains secure.

System Administrator

As a System Administrator, I want LLM interactions to estimate token costs and use early exit strategies, so that I can manage API expenses and optimize processing time.

Linguistic Researcher

As a Linguistic Researcher, I want SpeechPatternExtractor to fully implement advanced analysis methods (e.g., MFCCs, robust pitch tracking), so that I can obtain highly accurate and detailed prosodic insights for text rewriting.
