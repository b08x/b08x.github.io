SFL Prompt Architect

A full-stack application for creating, managing, and executing AI prompts and workflows with a visual interface.

Project Overview

SFL Prompt Architect is a comprehensive, full-stack solution designed to streamline the creation, management, and execution of AI prompts and complex prompt workflows. It provides a centralized library for reusable prompts and a powerful "Prompt Lab" with a visual, node-based editor for constructing, testing, and running intricate chains of AI tasks.

The application is built with a modern technology stack, featuring a React frontend and a Node.js backend, all containerized with Docker for easy setup and deployment. It aims to empower developers, prompt engineers, and content creators by providing robust tools to harness the capabilities of large language models (LLMs) like Google's Gemini.

Key Goals

  • Centralize Prompts: Create a single source of truth for all AI prompts, making them easy to find, version, and reuse across different applications.
  • Visualize Workflows: Offer an intuitive, drag-and-drop canvas to build, visualize, and debug complex multi-step AI workflows.
  • Facilitate Experimentation: Allow users to quickly test different prompts, models, and parameters to optimize results.
  • Simplify Integration: Provide a stable API for executing prompts and workflows from other services.
  • Enhance Creativity: Include AI-powered wizards to assist users in generating novel prompts and workflow ideas.

Core Functionality

Prompt Library

The Prompt Library is the central hub for managing all individual prompts. It supports full CRUD (Create, Read, Update, Delete) functionality.

  • Create & Edit: A form allows users to define a new prompt with a title, description, the prompt text itself, and associated tags for categorization.
  • Search & Filter: Users can search for prompts by keywords or filter them by tags, making it easy to find the right prompt for any task.
  • Versioning (Implicit): While not a formal versioning system, updating a prompt allows for iterative improvement over time.
  • AI-Powered Prompt Wizard: A "Prompt Wizard" assists users in crafting high-quality, structured prompts. Instead of starting from a blank page, the wizard guides the user through a form with specific fields based on Systemic Functional Linguistics (SFL) principles. This includes:
    • SFL: Field: Defines the topic and task type (e.g., Backend system design).
    • SFL: Tenor: Sets the AI's persona and desired tone (e.g., Expert, Formal).
    • Domain Specifics: Provides context and key concepts (e.g., Ruby programming, DAGs).
    • Keywords: Adds relevant keywords for focus and retrieval (e.g., NLP, vector database).
    • Target Audience: Specifies the intended audience for the generated text (e.g., Beginners, Experts).
    This structured input allows the LLM to generate a more precise and contextually-aware prompt, which can then be further refined by the user.

Prompt Lab & Workflow Engine

The Prompt Lab is the most powerful feature of the application. It uses a node-based UI (powered by React Flow) to build and execute workflows.

  • Visual Workflow Editor: Drag and drop nodes onto a canvas to represent different tasks. Connect nodes to define the flow of data and execution.
  • Task Nodes: Each node represents a specific action, such as executing a prompt, transforming data, or making a conditional choice.
  • Data Flow: The output of one node can be used as the input for another, allowing for complex data manipulation and chaining of AI calls.
  • Real-time Execution: Run the entire workflow and see the status of each node (pending, running, success, error) in real-time.
  • Inspect Results: Click on any completed node to view its inputs and outputs, which is crucial for debugging and optimization.

    • Services: Docker & Docker Compose

Local Development Setup

To run SFL Prompt Architect on your local machine, you'll need Docker and Docker Compose installed. The entire environment is orchestrated via a single `docker-compose` command.

Prerequisites

  • Docker Desktop (or Docker Engine + Docker Compose)
  • Git for cloning the repository
  • A code editor like VS Code

Installation Steps

  1. Clone the repository: 
    git clone <repository_url>
cd SFLPromptArchitect
  2. Configure environment variables:

    The backend requires an API key for the Google Gemini service. Create a `.env` file in the `backend/` directory by copying the example file:

    cp backend/.env.example backend/.env

    Then, edit `backend/.env` and add your Google AI Studio API key:

    # .env in backend/
GOOGLE_API_KEY=your_api_key_here
  3. Build and run the services:

    From the root directory of the project, run the following command. This will build the Docker images for the frontend and backend, start the database, and run all services.

    docker-compose up --build

    The `--build` flag is only necessary the first time or when you've made changes to the Dockerfiles or dependencies.

  4. Access the application:

    Once the containers are up and running, you can access the application in your web browser:

    • Frontend: http://localhost:8080
    • Backend API: http://localhost:3000
  5. Stopping the application:

    To stop all running containers, press `Ctrl+C` in the terminal where `docker-compose` is running, and then run:

    docker-compose down

How to Use the Application

After setting up the project, you can start using its features through the web interface.

Managing Prompts

  1. Navigate to the main page (Prompt Library).
  2. Click the "New Prompt" button to open a modal form. Fill in the details and save.
  3. Use the search bar and tag filters to find existing prompts.
  4. Click the pencil icon on a prompt card to edit it, or the trash icon to delete it.
  5. Click the "Wizard" button to get AI assistance. Fill out the structured form to generate a detailed prompt.

Using the Prompt Lab

  1. Navigate to the "Prompt Lab" from the sidebar.
  2. Click "New Workflow" to create a blank canvas. Give your workflow a name and description.
  3. From the controls, drag a "Task Node" onto the canvas.
  4. Click the new node to configure it. You can select a prompt from your library to be executed by this task.
  5. Add more nodes and connect them by dragging from the circular handles on the sides of each node. This defines the execution order.
  6. In the "User Input" area, provide any initial data needed for the first node in your workflow.
  7. Click the "Run" button. The nodes will change color to indicate their status (blue for running, green for success, red for error).
  8. Once finished, click on any node to see the data it received as input and the result it produced.

API Reference

The backend exposes a RESTful API for managing prompts and workflows. The base URL is http://localhost:3000/api.

Prompts

  • GET /prompts: Fetches all prompts.
  • POST /prompts: Creates a new prompt.
  • GET /prompts/:id: Fetches a single prompt by its ID.
  • PUT /prompts/:id: Updates an existing prompt.
  • DELETE /prompts/:id: Deletes a prompt.
  • POST /prompts/wizard: Generates a prompt using an LLM based on structured SFL input.

Workflows

  • GET /workflows: Fetches all workflows.
  • POST /workflows: Creates a new workflow.
  • GET /workflows/:id: Fetches a single workflow by its ID.
  • PUT /workflows/:id: Updates an existing workflow.
  • DELETE /workflows/:id: Deletes a workflow.
  • POST /workflows/execute/:id: Executes a workflow.
  • POST /workflows/wizard: Generates a workflow structure using an LLM.

Models

  • GET /models: Fetches a list of available AI models.

Gemini Service

  • POST /gemini/generate: A direct endpoint to generate text from a prompt using the Gemini model.
