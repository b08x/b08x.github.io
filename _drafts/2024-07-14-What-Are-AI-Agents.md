They're programs, with instructions written plain English, which is then translated into machine readable code....

So, an "Agent" is the newest variant of a "Programming Function" or if you're into Object Oriented Programming, a "method"

A set of instructions, in modular format, that performs a specific task...

The benefit? In the long run, or the 'near far future'...less effort in the programming of dynamic workflows. Instead of a programmer spend intensive amounts of cognitive effort, the language model functions (agents) are provided instructions in plain english to then work out the logic...the logic, is calculated based on data store that contains many many, many annotated examples...it doesn't really learn as much connects tagged metadata in statistically probably order order based on programmed criteria. 

What's different is the way we do now. The essence of the process is still the same. If anything, it will add a layer of abstraction to the underlying programming language...as the Programmer, to be most effective, also possess knowledge of how grammatical syntax structures extract and compile the data to generate the programming language code.

So what this allows for now, is a person could potentially pursue an advanced, (I don't always say) skill set in linguistics and be an effective systems or software engineer. 

The expertise in linguistics allows them to manipulate the language models to a degree that would be required to establish a context for any sort of systems or software objective. And what they didn't know about the specifics of systems and software, their research and analysis skill set that would naturally come along with a pursuit in linguistics would allow them to compile or gain the necessary knowledge for an objective. 

So if the objective is dockerizing an application that will fit within an Azure ecosystem, for example, the linguist doesn't necessarily know how to dockerize anything. It might need maybe a crash course on the concept because it's a hard concept to conceptualize. 

As far as fitting in with the Azure ecosystem, they can easily look up the instructions on how to do that. That's what we used to do with Google. So that process hasn't changed at all. It's just much better.
