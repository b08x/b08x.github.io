Summary

On the command line, the main executable is run with an argument or flag. For example, to parse a file, run the executable with the argument "parse" and provide the file path.

This creates variables used to instantiate a new instance of the NowQL parser class, where the file is then instantiated as an item. This is done with Item.new(source_path).

Depending on the file type or extension, a specific parser class is invoked. For example, for a subtitle file (extensions SRT, VTT, or ASS), the ParseSubtitle class is instantiated. Depending on the specific file type, if it's an SRT, it's passed to the SRT method, which returns only the text of the subtitle file. Similarly, the WebVTT method returns just the text, excluding timestamps or other metadata.

The convert method, depending on the item extension, converts SRT files to VTT and vice versa, if needed (though the latter might not be necessary).

Before the loop, the files array can be mapped to create an item object for each file. This allows the "add to database" method or class to store the path, file name, extension, type, creation time, and modification time for each file in the database.

After importing the files, options like "parse" or "chunk" can be applied.
