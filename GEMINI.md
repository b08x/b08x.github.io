# GEMINI.md - Syncopated Notes Project Context

## Project Overview
**Syncopated Notes** is a digital garden and personal knowledge base built with a unique "React Island" architecture integrated into a Jekyll static site. It combines the SEO and speed of static generation with the interactivity of modern React components.

- **Primary Stack:** Jekyll (Ruby), React 19 (TypeScript), Tailwind CSS 4.x, esbuild.
- **Key Features:** Digital garden notes with bidirectional links, automated multi-page wiki system, **Liminal Deck presentation system**, interactive React components (video players, graphs, diagrams), and an IDE/editor aesthetic.
- **Architectural Pattern:** Island Architecture. Jekyll generates static HTML with `data-island` markers, which are then hydrated by React asynchronously.

## Building and Running

### Development
```bash
# Start both Jekyll and React component watcher (Recommended)
npm run dev

# Or run separately
npm run watch:js      # React component watcher (esbuild)
npm run dev:jekyll    # Jekyll development server with LiveReload
```
Access the site at `http://localhost:4000`.

### Production Build
```bash
# Build everything (React, Workers, CSS, Jekyll)
npm run build

# Individual build commands
npm run build:js      # Build React bundle (assets/js/dist/garden-widgets-v2.js)
npm run build:css     # Build Tailwind CSS (assets/css/compiled.css)
npm run build:jekyll  # Build static site (_site/)
```

## Core Architecture: React Islands

Components are located in `src/components/` and must be registered in `src/main.tsx` to be hydrated.

### How to Add a Component
1.  **Create:** Add a new `.tsx` file in `src/components/`.
2.  **Register:** Add it to the `components` registry in `src/main.tsx` using `React.lazy()`.
3.  **Embed:** In any Markdown or HTML file, use the following syntax:
    ```html
    <div data-island="MyComponentName" data-props='{"prop1": "value"}'></div>
    ```
    *Note: Props can be raw JSON or Base64-encoded JSON for complex data.*

### Hydration & Auto-Enhancement
The system automatically enhances standard Markdown elements into React islands:
- **Code Blocks:** `div.highlighter-rouge` → `CodeBlock`
- **Mermaid Diagrams:** `img.mermaid` → `MermaidViewer`
- **Images:** `picture[data-lightbox="true"]` → `ImageLightbox`

**New Hydration Features:**
- **Async Event Queue:** Prevents hydration race conditions.
- **Mismatch Prevention:** Defensive hydration system for high-complexity components.

## Content Management

### 1. Digital Garden Notes (`_notes/`)
- Markdown files with YAML front matter.
- **Bidirectional Links:** Use `[[Note Title]]` syntax. Backlinks are automatically generated by the `bidirectional_links_generator.rb` plugin.
- **Callouts:** Supports Obsidian-style callouts (`> [!info]`).

### 2. Wiki System (`_data/wikis/`)
- **Source Data:** JSON files in `_data/wikis/`.
- **Generation:** The `wiki_page_generator.rb` plugin automatically creates paginated index pages and individual detail pages in `_wikis/`.
- **CRITICAL:** **NEVER** edit files in `_wikis/` directly. They are overwritten on every build. Modify the JSON source instead.

### 3. Liminal Deck System (`_slides/`)
- **Concept:** High-impact, macro-typography presentation system.
- **Source:** Markdown files in `_slides/` organized by deck ID.
- **Generation:** Uses `_layouts/liminal-deck.html` and `LiminalDeckIsland` for interactive delivery.
- **Features:** Fragment support, touch/wheel navigation, progress tracking.
- **Wizard:** Use `scripts/liminal_wizard.rb` to generate new decks via LLM.

## Development Conventions

### Theming & Styling
- **Aesthetic:** High-contrast IDE/Editor theme. Mono fonts for UI, Serif for prose.
- **Styling:** Use Tailwind CSS 4.x utilities.
- **Colors:** **NEVER** hardcode colors in React. Use CSS variables (e.g., `--foreground`, `--accent`, `--border`) defined in `_sass/_theme-variables.scss`.
- **Dark Mode:** Controlled by the `.dark` class on `<html>`. React components should use a `MutationObserver` on `document.documentElement` to sync theme state if they rely on JS-driven colors (like D3 or Mermaid).

### Custom Jekyll Plugins (`_plugins/`)
- `wiki_page_generator.rb`: Main wiki logic.
- `bidirectional_links_generator.rb`: Handles `[[wikilinks]]`.
- `obsidian_callouts.rb`: Converts Obsidian syntax to HTML.
- `render_liquid.rb`: Allows Liquid tags inside JSON data/strings.

## Project Structure Map
- `_data/wikis/`: Wiki source JSON.
- `_includes/`: Reusable Liquid partials.
- `_layouts/`: Jekyll templates (Primary: `sidebar-layout.html`, `liminal-deck.html`).
- `_notes/`: Digital garden Markdown content.
- `_slides/`: Liminal Deck slide content.
- `pages/`: Static pages (formerly `_pages/`).
- `_plugins/`: Custom Ruby extensions.
- `_sass/`: SCSS theme variables and styles.
- `scripts/`: Tooling and automation (e.g., `liminal_wizard.rb`).
- `assets/js/dist/`: Compiled JS bundles (ignored by git).
- `src/components/`: React island source files.
- `src/main.tsx`: Component registry and hydration logic.

---

## Anti-Patterns (What to Avoid)
- **NO Direct Edits to `_wikis/`:** Always update `_data/wikis/*.json`.
- **NO Synchronous Imports in `main.tsx`:** Always use `React.lazy()` for island components to keep the initial bundle small.
- **NO Hardcoded Colors:** Use CSS variables for theme compatibility.
- **NO Script Tags in Markdown:** Use React islands for interactivity.
- **NO Wiki Slugs = "page":** Avoid using "page" as a wiki entry slug (auto-suffixed to `-content` to avoid conflicts).
